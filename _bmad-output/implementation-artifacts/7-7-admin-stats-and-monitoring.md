# Story 7.7: Просмотр операционных сигналов и статистики (Admin)

Status: done

## Story

As an operator,
I want видеть ключевые метрики (количество сессий, активные подписки, ошибки) через Telegram,
So that я могу быстро оценивать состояние системы.

## Acceptance Criteria

1. **Given** нажатие кнопки "📊 Статистика" в админ-меню, **When** запрос обрабатывается, **Then** бот должен отправить сводную статистику. [Source: epics.md#story-7.7-AC1]

2. **Given** запрос статистики, **When** данные агрегируются, **Then** ответ должен включать:
    - Общее количество пользователей.
    - Общее количество сессий.
    - Количество активных и просроченных подписок.
    - Статистику за последние 24 часа (новые сессии, успешные оплаты). [Source: epics.md#story-7.7-AC2]

3. **Given** расчет метрик, **When** база данных содержит данные, **Then** агрегация должна быть корректной и учитывать временные зоны (UTC). [Source: architecture.md#data-integrity]
