# Story 4.5: Просмотр статуса подписки и даты продления

Status: done

## Story

As a пользователь,
I want видеть актуальную информацию о моей подписке (статус и дату следующего списания),
So that я могу планировать свои расходы и использование сервиса.

## Acceptance Criteria

1. **Given** запрос `/status`, **When** у пользователя есть активная подписка, **Then** бот должен отправить сообщение "Ваша подписка активна. Следующее списание: DD.MM.YYYY". [Source: epics.md#story-4.5-AC1]

2. **Given** статус `past_due`, **When** пользователь запрашивает статус, **Then** сообщение должно информировать о льготном периоде и указывать оставшееся время в часах. [Source: epics.md#story-4.5-AC2]

3. **Given** статус `suspended`, **When** пользователь запрашивает статус, **Then** сообщение должно сообщать о приостановке доступа с кнопкой для повторного оформления. [Source: epics.md#story-4.5-AC3]

4. **Given** отсутствие подписки, **When** пользователь запрашивает статус, **Then** система должна показывать стандартный статус для бесплатного тарифа (как в Story 4.1). [Source: architecture.md#continuity]
