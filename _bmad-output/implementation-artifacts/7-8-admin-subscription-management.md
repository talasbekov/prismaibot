# Story 7.8: Управление подписками и разбор проблемных платежей

Status: done

## Story

As an operator,
I want иметь возможность проверить статус оплаты конкретного пользователя и вручную корректировать доступ,
So that я могу решать тикеты поддержки пользователей.

## Acceptance Criteria

1. **Given** администратор отправляет Telegram ID пользователя текстом, **When** сообщение обрабатывается, **Then** бот должен вернуть подробную информацию о биллинге этого пользователя. [Source: epics.md#story-7.8-AC1]

2. **Given** просмотр деталей пользователя, **When** у него нет Premium доступа, **Then** бот должен показать кнопку "🚀 Выдать Premium (30д)". [Source: epics.md#story-7.8-AC2]

3. **Given** нажатие кнопки "Выдать Premium", **When** запрос обрабатывается, **Then** система должна:
    - Установить `access_tier = "premium"`.
    - Создать/продлить подписку на 30 дней с типом `manual_admin`.
    - Уведомить пользователя о выдаче доступа. [Source: epics.md#story-7.8-AC3]

4. **Given** поиск пользователя, **When** ID не существует в базе, **Then** бот должен сообщить, что пользователь не найден. [Source: architecture.md#error-handling]
